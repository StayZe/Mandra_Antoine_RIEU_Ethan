EXERCICE 1 : ANALYSE DE COMPLEXITE

1. COMPLEXITE EN MEILLEUR/MOYEN/PIRE CAS

Code 1 : Double boucle
for(int i = 0; i < n; i++) {
    for(int j = 0; j < n; j++) {
        printf("%d", arr[i][j]);
    }
}

Meilleur cas : O(n²)
Cas moyen : O(n²)
Pire cas : O(n²)
Espace mémoire : O(1)
Explication : Deux boucles imbriquées

Code 2 : Boucle logarithmique
for(int i = 1; i < n; i *= 2) {
    printf("%d", arr[i]);
}

Meilleur cas : O(log n)
Cas moyen : O(log n)
Pire cas : O(log n)
Espace mémoire : O(1)
Explication : i double à chaque itération

Code 3 : Récursion logarithmique
void recursive(int n) {
    if(n <= 1) return;
    printf("%d", n);
    recursive(n/2);
}

Meilleur cas : O(log n)
Cas moyen : O(log n)
Pire cas : O(log n)
Espace mémoire : O(log n)
Explication : n divisé par 2 à chaque appel récursif


2. IMPACT POUR n = 1 000 000

Code 1 (Double boucle)
Opérations : 1,000,000,000,000
